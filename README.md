# LTC6904-Programmable-Oscillator-Control
This project provides a complete solution for controlling the LTC6904 programmable oscillator using an ESP32 via the I2C protocol. It is specifically designed for radio frequency (RF) applications, such as replacing crystals in vintage radios or generating precise clock signals from 1 kHz to 68 MHz.

🛠️ Hardware Overview & Pinout1. LTC6904 (I2C Version)The LTC690X family comes in two flavors: the LTC6903 (SPI) and the LTC6904 (I2C). This project is built for the I2C version.LTC6904 PinNameConnection to ESP32 / Circuit1GNDCommon Ground2AD2GND (Sets I2C Address to 0x17)3SCLGPIO 22 (ESP32)4SDAGPIO 21 (ESP32)5AD1GND (Sets I2C Address to 0x17)6CLKRF Signal Output7CLK InvNo Connection (or Inverted Output)8VCC3.3V DC (Warning: Do NOT use 5V)2. ESP32 WiringSDA: GPIO 21SCL: GPIO 22Power: 3.3V Pin from ESP32 to Pin 8 of LTC6904.🔌 Circuit Implementation (Filtering & Components)Because the LTC6904 generates a square wave, the output contains high-frequency harmonics. For RF and radio applications (like a BFO), signal conditioning is required:VCC Decoupling: Place a 0.1µF (100nF) ceramic capacitor between Pin 8 (VCC) and Pin 1 (GND) as close to the chip as possible to reduce digital noise.DC Blocking: Connect a 470pF capacitor in series with the Output (Pin 6). This removes the DC offset.Impedance Matching/Attenuation: Add a 4.7kΩ resistor in series after the capacitor before injecting the signal into the radio's circuit. This prevents overloading the receiver's mixer.🖥️ The Calculator App (HTML Utility)The LTC6904 does not accept "MHz" values directly. It requires two 8-bit bytes containing a specific Octave (4 bits) and DAC value (10 bits).How to generate values:Open the provided LTC6904_Master_Calc.html in any web browser.Enter your target frequency in MHz (e.g., 12.697).The app will output the MSB and LSB Hexadecimal values.Example for 12.697 MHz: MSB: 0xCA, LSB: 0xB5.💻 ESP32 Source Code (Arduino IDE)Copy and paste this code into your Arduino IDE. Update the regMSB and regLSB variables with the values generated by the calculator.C++#include <Wire.h>

// Default I2C Address (AD1 and AD2 tied to GND)
const int LTC_ADDR = 0x17; 

// -------------------------------------------------------
// VALUES GENERATED BY THE HTML CALCULATOR
// -------------------------------------------------------
byte regMSB = 0xCA;  // Replace with your generated MSB
byte regLSB = 0xB5;  // Replace with your generated LSB
// -------------------------------------------------------

void setup() {
  // Initialize I2C (SDA 21, SCL 22)
  Wire.begin(21, 22); 
  Serial.begin(115200);
  
  delay(1000);
  
  Serial.println("LTC6904 Programmer Initialized...");
  Serial.print("Sending MSB: 0x"); Serial.println(regMSB, HEX);
  Serial.print("Sending LSB: 0x"); Serial.println(regLSB, HEX);
  
  // Start Transmission
  Wire.beginTransmission(LTC_ADDR);
  Wire.write(regMSB); 
  Wire.write(regLSB);
  
  byte status = Wire.endTransmission();
  
  if (status == 0) {
    Serial.println(">>> SUCCESS: LTC6904 Programmed successfully!");
  } else {
    Serial.print(">>> ERROR: I2C Transmission failed. Code: ");
    Serial.println(status);
  }
}

void loop() {
  // The LTC6904 is non-volatile as long as it has power. 
  // The ESP32 does not need to keep sending data.
}
📝 Important NotesI2C Address: If you leave AD1 or AD2 floating or tied to VCC, the address will change. Ensure both are tied to GND to use 0x17.Heat & Drift: The LTC6904 is an all-silicon oscillator. It may drift slightly as the chip warms up. For high-stability needs, keep it away from power transistors or hot components inside the radio.Square Wave Harmonics: If you experience "ghost signals" in your radio, increase the value of the series resistor or add a simple low-pass filter to the output.
